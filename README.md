Личный финансовый менеджер

Консольное приложение для управления личными финансами на Java.

Функциональность

Учет доходов и расходов по категориям

Установка бюджетов по категориям

Многопользовательская система с авторизацией

Переводы между пользователями

Статистика и аналитика расходов

Сохранение данных между сеансами

Архитектура

Основные классы:

PersonalFinanceApp - главный класс, точка входа

FinanceManager - управляет пользователями и авторизацией

User - представляет пользователя системы

Wallet - кошелек пользователя (центральный класс)

Transaction - финансовая операция (доход/расход)

Budget - бюджет по категории расходов

Структура данных:

FinanceManager → Map<User> → Wallet → List<Transaction>, List<Budget>

Быстрый старт

Требования:

Java 8 или выше

Командная строка или IDE

Запуск:

Скомпилируйте все файлы:

javac *.java

Запустите приложение:

java PersonalFinanceApp

Сохранение данных
Данные автоматически сохраняются:

При выходе из системы

При переводе средств

При завершении приложения

Файлы хранятся в папке wallets/:

wallets/wallet_username.dat - данные кошелька

users.dat - список пользователей

Основные команды
В меню авторизации:
1 - Войти в систему

2 - Зарегистрироваться

3 - Показать список пользователей

4 - Выйти из приложения

В главном меню:
1 - Добавить доход

2 - Добавить расход

3 - Установить/изменить бюджет

4 - Показать статистику

5 - Показать информацию о бюджетах

6 - Показать историю транзакций

7 - Перевести средства другому пользователю

8 - Сохранить данные

9 - Выйти из системы

0 - Сохранить и выйти из приложения

Файловая структура проекта

PersonalFinanceApp/

├── PersonalFinanceApp.java    # Главный класс приложения

├── FinanceManager.java        # Менеджер пользователей и авторизации

├── User.java                  # Класс пользователя

├── Wallet.java                # Класс кошелька

├── Transaction.java           # Класс транзакции

├── Budget.java                # Класс бюджета

├── wallets/                   # Папка с сохраненными данными

│   ├── wallet_user1.dat      # Данные кошелька user1

│   └── wallet_user2.dat      # Данные кошелька user2

└── users.dat                  # Файл с пользователями

Особенности реализации
Все данные хранятся в памяти во время работы приложения

При выходе данные сериализуются в файлы

При входе данные загружаются из файлов

Валидация пользовательского ввода

Уведомления о превышении бюджета

Поддержка нескольких пользователей
